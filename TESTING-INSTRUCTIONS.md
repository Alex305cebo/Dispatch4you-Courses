# 🧪 ИНСТРУКЦИЯ ПО ТЕСТИРОВАНИЮ УЛУЧШЕНИЙ

## Что нужно протестировать:

### 1. АВТОМАТИЧЕСКАЯ ПРОВЕРКА ДИАЛОГОВ
- [ ] Открыть страницу simulator.html
- [ ] Открыть консоль (F12)
- [ ] Проверить что выводится: "🔍 Проверка всех диалогов..."
- [ ] Проверить что все диалоги проверены
- [ ] Если есть ошибки - должен быть alert

### 2. ВИЗУАЛЬНЫЕ ИНДИКАТОРЫ
- [ ] Начать любой диалог
- [ ] Выбрать любой вариант ответа
- [ ] Должен появиться индикатор пути (💎 МАСТЕР ПУТЬ или другой)
- [ ] Индикатор должен быть цветным

### 3. FALLBACK СИСТЕМА
- [ ] Начать диалог LA → Chicago (🧪 EXPERT TEST)
- [ ] Выбрать "good" вариант (средний)
- [ ] Проверить что диалог продолжается (не застревает)
- [ ] Может появиться уведомление о fallback

### 4. ВОЗВРАТ НАЗАД
- [ ] Начать любой диалог
- [ ] Выбрать вариант ответа
- [ ] Должна появиться кнопка "↩️ Вернуться назад" (справа внизу)
- [ ] Нажать на кнопку
- [ ] Проверить что вернулись на предыдущий шаг
- [ ] Варианты ответов должны появиться снова

### 5. РЕЖИМ ОБУЧЕНИЯ vs ТЕСТИРОВАНИЯ
- [ ] Проверить что есть кнопка режима (справа вверху)
- [ ] По умолчанию: "📚 Обучение" (зеленая)
- [ ] Нажать на кнопку
- [ ] Должен переключиться на "🎯 Тестирование" (красная)
- [ ] Начать диалог в режиме тестирования
- [ ] Кнопка "Вернуться назад" должна быть скрыта
- [ ] Переключить обратно на обучение
- [ ] Кнопка "Вернуться назад" должна появиться

---

## Если что-то не работает:

1. Откройте консоль (F12)
2. Проверьте ошибки JavaScript
3. Проверьте что файл scenarios-data-v2.js загружен
4. Обновите страницу с очисткой кэша (Ctrl+Shift+R)
