# Инструкция по перезагрузке изменений

## Проблема

Изменения в коде не применились потому что Metro bundler закэшировал старую версию.

## Решение

### Вариант 1: Полная перезагрузка (рекомендуется)

```bash
# 1. Останови Metro bundler (Ctrl+C в терминале где он запущен)

# 2. Очисти кэш
cd adventure
npm start -- --reset-cache

# 3. После запуска Metro bundler:
# - Нажми 'r' чтобы перезагрузить
# - Или нажми 'w' чтобы открыть в браузере заново
```

### Вариант 2: Быстрая перезагрузка

Если Metro bundler уже запущен:

```bash
# В терминале где запущен Metro bundler нажми:
# Shift + R  (полная перезагрузка с очисткой кэша)
```

### Вариант 3: Через браузер

1. Открой DevTools (F12)
2. Нажми правой кнопкой на кнопку "Обновить"
3. Выбери "Очистить кэш и жёсткая перезагрузка"

Или:
- Windows/Linux: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

## Проверка что изменения применились

После перезагрузки:

1. Открой Load Board
2. Выбери любой груз
3. Нажми "📞 Позвонить брокеру"
4. Проведи переговоры до успеха

**Ожидаемый результат:**
- После успешных переговоров показывается **карточка выбранного трака**
- НЕ показывается отдельное окно "Назначить водителя"
- Есть кнопка "Изменить" (если траков больше 1)
- Есть кнопка "✅ Подтвердить"

**Если всё ещё показывается старое окно:**
- Значит кэш не очистился
- Попробуй Вариант 1 (полная перезагрузка)

## Дополнительные шаги

### Если ничего не помогает:

```bash
# 1. Останови Metro bundler (Ctrl+C)

# 2. Удали кэш вручную
cd adventure
rm -rf node_modules/.cache
rm -rf .expo

# 3. Перезапусти
npm start -- --reset-cache
```

### Проверка версии файлов

Убедись что файлы изменены:

```bash
cd adventure

# Проверь что NegotiationChat.tsx содержит новый код
grep "preselectedTruckId" components/NegotiationChat.tsx

# Должно вывести:
# preselectedTruckId?: string | null;

# Проверь что LoadBoardPanel.tsx не использует AssignModal
grep "AssignModal" components/LoadBoardPanel.tsx

# Не должно ничего найти
```

## Логи для отладки

Открой консоль браузера (F12) и проверь:

```
# Должны быть логи:
✅ Best truck selected: T1 (score: 125.5)
✅ Truck T1 assigned to load INIT-L1

# НЕ должно быть:
❌ AssignModal opened
❌ pendingLoad set
```

## Частые проблемы

**Q: Metro bundler не запускается**
```bash
# Убей все процессы Node.js
killall node  # Mac/Linux
taskkill /F /IM node.exe  # Windows

# Перезапусти
npm start
```

**Q: Ошибка "Port 8081 already in use"**
```bash
# Останови процесс на порту 8081
lsof -ti:8081 | xargs kill  # Mac/Linux
netstat -ano | findstr :8081  # Windows (найди PID)
taskkill /PID <PID> /F  # Windows (убей процесс)
```

**Q: Изменения применились частично**
```bash
# Полная очистка
cd adventure
rm -rf node_modules/.cache
rm -rf .expo
rm -rf dist
npm start -- --reset-cache
```

## Контрольный чеклист

- [ ] Metro bundler остановлен
- [ ] Кэш очищен (`--reset-cache`)
- [ ] Metro bundler перезапущен
- [ ] Браузер перезагружен (Ctrl+Shift+R)
- [ ] Консоль не показывает ошибок
- [ ] Новый UI показывается после переговоров
- [ ] Старое окно "Назначить водителя" НЕ показывается

## Если всё работает

После успешной перезагрузки ты должен увидеть:

1. **Во время переговоров:**
   - Окно чата с брокером
   - Кнопки выбора ставки
   - Никаких дополнительных окон

2. **После успешных переговоров:**
   - Карточка выбранного трака
   - Информация о траке (HOS, настроение, deadhead)
   - Кнопка "Изменить" (если траков > 1)
   - Кнопка "✅ Подтвердить"
   - Кнопка "Отмена"

3. **После подтверждения:**
   - Окно закрывается
   - Трак начинает движение к погрузке
   - Груз появляется в "My Loads"
