# 💾 Система сохранения игры

## Обзор
Игра автоматически сохраняет прогресс в localStorage браузера. При перезагрузке страницы игра продолжается с того же места.

## Что сохраняется

### Основное состояние игры
- **phase** — текущая фаза (menu, playing, shift-end, finish)
- **day** — текущий день
- **gameMinute** — игровое время (минуты с начала смены)

### Финансы
- **balance** — текущий баланс
- **totalEarned** — всего заработано
- **totalLost** — всего потрачено
- **financeLog** — история транзакций

### Репутация
- **reputation** — общая репутация (0-100)
- **brokers** — отношения с брокерами

### Траки
- **trucks** — все траки с их состоянием:
  - Позиция на карте
  - Текущий груз
  - HOS часы
  - Маршрут
  - Статус (idle, driving, loaded, etc.)
  - Метрики производительности

### Грузы
- **availableLoads** — доступные грузы на Load Board
- **activeLoads** — грузы в работе
- **bookedLoads** — забуканные грузы (ожидают назначения)

### События и уведомления
- **activeEvents** — активные события
- **resolvedEvents** — решённые события
- **notifications** — все уведомления
- **unreadCount** — количество непрочитанных

## Что НЕ сохраняется

Временные UI состояния (сбрасываются при перезагрузке):
- **negotiation** — открытое окно переговоров
- **selectedTruckId** — выбранный трак
- **selectedEventId** — выбранное событие
- **selectedLoadId** — выбранный груз на карте

Это сделано намеренно, чтобы при перезагрузке игрок начинал с чистого UI.

## Технические детали

### Zustand Persist Middleware
```typescript
import { persist, createJSONStorage } from 'zustand/middleware';

export const useGameStore = create<GameState>()(
  persist(
    (set, get) => ({ /* state */ }),
    {
      name: 'dispatcher-game-storage',
      storage: createJSONStorage(() => localStorage),
      partialize: (state) => ({ /* что сохранять */ }),
    }
  )
);
```

### Ключ в localStorage
`dispatcher-game-storage`

### Формат данных
JSON с полным состоянием игры

## Как это работает

1. **Автоматическое сохранение**
   - При любом изменении состояния
   - Мгновенно в localStorage
   - Без задержек

2. **Автоматическая загрузка**
   - При открытии страницы
   - Проверяет наличие сохранения
   - Восстанавливает состояние

3. **Гидратация**
   - Zustand автоматически восстанавливает состояние
   - Все компоненты получают сохранённые данные
   - Игра продолжается с того же места

## Управление сохранениями

### Очистка сохранения
```javascript
// В консоли браузера
localStorage.removeItem('dispatcher-game-storage');
```

### Просмотр сохранения
```javascript
// В консоли браузера
JSON.parse(localStorage.getItem('dispatcher-game-storage'));
```

### Экспорт сохранения
```javascript
// В консоли браузера
const save = localStorage.getItem('dispatcher-game-storage');
console.log(save);
// Скопировать и сохранить в файл
```

### Импорт сохранения
```javascript
// В консоли браузера
const save = '...'; // вставить сохранение
localStorage.setItem('dispatcher-game-storage', save);
location.reload();
```

## Размер сохранения

Примерный размер: **50-200 KB**

Включает:
- 3 трака с полными данными
- ~20 грузов
- ~50 уведомлений
- История транзакций
- События и метрики

## Ограничения localStorage

- **Максимум**: 5-10 MB (зависит от браузера)
- **Наше использование**: ~0.2 MB
- **Запас**: 25-50x

## Совместимость

Работает во всех современных браузерах:
- Chrome/Edge
- Firefox
- Safari
- Opera

## Приватность

- Данные хранятся **только локально**
- Не отправляются на сервер
- Не синхронизируются между устройствами
- Удаляются при очистке данных браузера

## Отладка

### Проверка работы сохранения
1. Начать игру
2. Сделать несколько действий
3. Перезагрузить страницу (F5)
4. Убедиться что состояние восстановилось

### Проблемы и решения

**Проблема**: Сохранение не работает
- Проверить: localStorage доступен?
- Проверить: приватный режим браузера?
- Проверить: квота не превышена?

**Проблема**: Старое сохранение конфликтует
- Решение: Очистить localStorage
- Или: Изменить версию ключа

**Проблема**: Данные повреждены
- Решение: Очистить и начать заново
- Или: Восстановить из резервной копии

## Миграция версий

При обновлении структуры данных:
1. Изменить ключ: `dispatcher-game-storage-v2`
2. Добавить миграцию старых данных
3. Или: Предложить начать новую игру

## Безопасность

- Не храним чувствительные данные
- Только игровой прогресс
- Можно безопасно делиться сохранениями
- Нет риска для пользователя
