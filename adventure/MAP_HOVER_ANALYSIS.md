# Анализ: Почему hover штатов не работает

## Проблема

Попытки добавить интерактивность (подсветку при наведении) на штаты карты не работают. Изменения в коде либо не применяются, либо создают ошибки компиляции.

## Причины

### 1. Архитектура Canvas
Текущая карта использует Canvas API с постоянной перерисовкой через `requestAnimationFrame`. Это создаёт проблемы:

- **Нет автоматической перерисовки при изменении state** — useEffect не имеет зависимости от `hoveredState`
- **Ref не триггерит перерисовку** — `hoveredStateRef.current` меняется, но Canvas не перерисовывается
- **Нужна зависимость в useEffect** — но это создаст бесконечный цикл перерисовок

### 2. Проблемы с файловой системой
- `fsWrite` не применяет изменения — файл возвращается к старой версии
- `strReplace` создаёт дубликаты кода и ломает синтаксис
- Dev сервер кэширует старую версию файла
- Git показывает изменения, но они не видны в редакторе

### 3. Координатная трансформация
Для проверки попадания курсора в штат нужно:
1. Получить координаты мыши относительно canvas
2. Применить обратную трансформацию (scale + translate)
3. Проверить `ctx.isPointInPath()` для каждого штата
4. Сохранить результат в state
5. Перерисовать canvas с новым цветом

Но шаг 5 не происходит автоматически!

## Решение

### Вариант 1: Добавить зависимость в useEffect (РИСКОВАННО)

```typescript
useEffect(() => {
  // ... draw function
  function loop() {
    draw(pulseRef.current);
    animRef.current = requestAnimationFrame(loop);
  }
  loop();
  return () => cancelAnimationFrame(animRef.current);
}, [hoveredState]); // ← добавить зависимость
```

**Проблема**: Может создать бесконечный цикл или лаги.

### Вариант 2: Перерисовка только при hover (ОПТИМАЛЬНО)

```typescript
const [hoveredState, setHoveredState] = useState<string | null>(null);

// Отдельный useEffect для hover
useEffect(() => {
  if (!canvasRef.current) return;
  // Перерисовать один раз при изменении hoveredState
  draw(pulseRef.current);
}, [hoveredState]);
```

### Вариант 3: Использовать SVG вместо Canvas

SVG поддерживает hover нативно через CSS:

```tsx
<svg>
  {states.map(state => (
    <path
      d={state.path}
      className="state"
      onMouseEnter={() => setHovered(state.name)}
    />
  ))}
</svg>

<style>
.state:hover {
  fill: #2d6a4f;
  stroke: #4ade80;
}
</style>
```

**Плюсы**: Работает из коробки, нет проблем с перерисовкой
**Минусы**: Нужно переписать всю карту

### Вариант 4: Оставить как есть

Карта работает, показывает траки, маршруты, статусы. Hover штатов — nice-to-have, но не критично для игры.

## Рекомендация

Оставить карту как есть. Интерактивность штатов не является критичной функцией для игры. Фокус лучше направить на:
- Механику игры
- Баланс экономики
- Контент (события, грузы, документы)
- Полировку UI других компонентов

Если всё же нужна интерактивность — использовать Вариант 2 (отдельный useEffect для hover).
