# 📧 Система Email — Полное руководство

## Обзор
Полноценная система электронной почты для коммуникации с брокерами, отправки документов и решения вопросов.

## Возможности

### 1. Чтение писем
- Входящие письма от брокеров
- Фильтры: Все / Непрочитанные / Важные
- Детальный просмотр письма
- Автоматическая пометка как прочитанное

### 2. Написание новых писем
**Кнопка "✉️ Написать"** в правом верхнем углу

#### Шаблоны писем:
1. **💰 Запрос ставки** — узнать цены на маршрут
2. **📄 Отправка POD** — отправить подписанный документ
3. **⏱️ Запрос Detention** — запросить оплату за ожидание
4. **🚛 Обновление по грузу** — сообщить статус доставки
5. **📋 Rate Con подписан** — подтвердить подписание контракта
6. **⚠️ Сообщить о проблеме** — сообщить о проблеме с грузом

#### Процесс создания письма:
1. Выбрать получателя (брокера)
2. Выбрать шаблон или написать с нуля
3. Отредактировать тему и текст
4. Использовать быстрые фразы:
   - "+ Thank you"
   - "+ Please confirm"
   - "+ Looking forward"
5. Отправить

### 3. Ответы на письма
**Кнопка "↩️ Ответить"** в детальном просмотре письма

- Автоматическое заполнение получателя
- Тема с префиксом "Re:"
- Цитирование оригинального сообщения
- Быстрые фразы для ответов

## Типы писем

### Входящие от брокеров:
- **📧 Email** — обычные письма
- **📄 POD Ready** — запрос на отправку POD
- **📋 Rate Con** — контракт на подпись
- **⏱️ Detention** — вопросы по ожиданию

### Исходящие:
- Запросы ставок
- Отправка документов
- Обновления статуса
- Решение проблем

## Автоматические ответы

Брокеры отвечают через 2-4 минуты игрового времени:
- "Got it, thanks for the update!"
- "Received. Will process shortly."
- "Thank you! Payment will be processed within 24 hours."
- "Confirmed. Driver can proceed."
- "Approved. Will send updated documents."

## Уведомления

После отправки письма:
- ✓ Подтверждение отправки
- 📧 Ответ брокера (через несколько минут)

## Функции в gameStore

### sendEmail()
```typescript
sendEmail({
  to: string,           // Имя брокера
  subject: string,      // Тема письма
  body: string,         // Текст письма
  isReply: boolean,     // Это ответ?
  replyToId?: string    // ID оригинального письма
})
```

### Другие функции:
- `addNotification()` — добавить уведомление
- `markNotificationRead()` — пометить как прочитанное
- `markAllNotificationsRead()` — пометить все
- `clearNotifications()` — очистить все

## Советы по использованию

### Когда писать письма:
1. **После доставки** — отправить POD
2. **При задержке** — запросить detention
3. **Проблемы** — сообщить о проблемах
4. **Обновления** — держать брокера в курсе
5. **Новые грузы** — запросить ставки

### Профессиональная коммуникация:
- Используйте шаблоны для стандартных ситуаций
- Будьте кратки и по делу
- Всегда благодарите ("Thank you!")
- Запрашивайте подтверждение ("Please confirm")
- Держите брокера в курсе статуса

### Приоритеты:
- 🚨 **СРОЧНО** — ответить немедленно
- ⚡ **Требуется ответ** — ответить в течение часа
- 📧 **Обычные** — ответить когда удобно

## Реализм

В реальной индустрии:
- Email — основной канал коммуникации
- Быстрые ответы = хорошая репутация
- Документация всех договоренностей
- Профессиональный тон обязателен
- Своевременная отправка POD критична для оплаты
