# Тестирование системы выбора трака

## Как протестировать

### 1. Запуск игры

```bash
cd adventure
npm start
```

### 2. Начало новой смены

1. Открой игру
2. Создай новую сессию (3-5 траков)
3. Перейди на вкладку "Load Board"

### 3. Тест автоматического выбора

**Сценарий 1: Один свободный трак**
1. Найди груз на Load Board
2. Нажми "📞 Позвонить брокеру"
3. Проведи переговоры (любая ставка)
4. После успешной сделки — проверь:
   - ✅ Показана карточка выбранного трака
   - ✅ Трак ближайший к точке погрузки
   - ✅ Показан deadhead (если > 0)
   - ✅ Показаны HOS и настроение

**Сценарий 2: Несколько свободных траков**
1. Убедись что есть 2+ свободных трака
2. Начни переговоры по грузу
3. После сделки — проверь:
   - ✅ Выбран оптимальный трак (ближайший)
   - ✅ Есть кнопка "Изменить"
4. Нажми "Изменить":
   - ✅ Открылся список всех доступных траков
   - ✅ У каждого показан deadhead
   - ✅ Выбранный трак отмечен галочкой ✓
5. Выбери другой трак:
   - ✅ Список закрылся
   - ✅ Карточка обновилась

**Сценарий 3: Prebook (трак на разгрузке)**
1. Найди трак со статусом `at_delivery`
2. Начни переговоры по грузу
3. После сделки выбери этот трак:
   - ✅ Показан статус "🔄 Заканчивает разгрузку"
   - ✅ Можно подтвердить назначение
4. Подтверди:
   - ✅ Груз назначен как "следующий"
   - ✅ Трак продолжает разгрузку

**Сценарий 4: Нет свободных траков**
1. Убедись что все траки заняты (в пути)
2. Открой Load Board:
   - ✅ Кнопка "📞 Позвонить брокеру" неактивна
   - ✅ Показано "🚫 Нет свободных траков"

### 4. Тест подтверждения

1. Выбери трак после переговоров
2. Нажми "✅ Подтвердить":
   - ✅ Окно переговоров закрылось
   - ✅ Трак начал движение к погрузке
   - ✅ На карте видно движение трака
   - ✅ В списке "My Loads" появился груз

### 5. Тест отмены

1. После успешных переговоров
2. Нажми "Отмена":
   - ✅ Окно закрылось
   - ✅ Груз НЕ назначен
   - ✅ Трак остался в прежнем статусе

## Проверка расчёта deadhead

### Пример 1: Близкий трак
- Трак в Chicago
- Груз из Chicago → Dallas
- **Ожидаемый deadhead:** 0 миль

### Пример 2: Дальний трак
- Трак в Miami
- Груз из Chicago → Dallas
- **Ожидаемый deadhead:** ~1377 миль

### Пример 3: Средний deadhead
- Трак в Indianapolis
- Груз из Chicago → Dallas
- **Ожидаемый deadhead:** ~180 миль

## Проверка приоритетов

**Тест: Два трака, разный deadhead**
- Трак A: Chicago, HOS 11ч, mood 70%
- Трак B: Miami, HOS 11ч, mood 70%
- Груз: Chicago → Dallas
- **Ожидается:** Выбран Трак A (deadhead 0 vs 1377)

**Тест: Два трака, разный HOS**
- Трак A: Chicago, HOS 5ч, mood 70%
- Трак B: Chicago, HOS 11ч, mood 70%
- Груз: Chicago → Dallas
- **Ожидается:** Выбран Трак B (больше HOS)

**Тест: Idle vs At Delivery**
- Трак A: idle, Chicago, HOS 11ч
- Трак B: at_delivery, Chicago, HOS 11ч
- Груз: Chicago → Dallas
- **Ожидается:** Выбран Трак A (idle bonus +50)

## Известные проблемы

- [ ] Если груз истекает через < 10 минут — может не успеть
- [ ] Deadhead > 500 миль — может быть невыгодно
- [ ] Prebook не учитывает время завершения разгрузки
