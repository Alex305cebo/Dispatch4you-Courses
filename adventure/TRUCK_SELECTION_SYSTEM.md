# Система выбора трака для груза

## Обзор

Новая система автоматического выбора трака интегрирована прямо в окно переговоров с брокером. Игроку больше не нужно выбирать трак после успешных переговоров — система автоматически предлагает лучший вариант.

## Как это работает

### 1. Автоматический выбор лучшего трака

При открытии переговоров система автоматически выбирает оптимальный трак по следующим критериям:

**Приоритеты выбора:**
1. **Deadhead (расстояние до погрузки)** — ближайший трак к точке погрузки получает преимущество
2. **HOS (Hours of Service)** — траки с большим запасом часов предпочтительнее
3. **Настроение водителя** — учитывается mood score (0-100)
4. **Статус трака** — idle траки получают бонус +50 баллов (готовы сразу)

**Формула расчёта:**
```typescript
totalScore = (hoursLeft * 10) + (mood / 10) + statusBonus - (deadhead * 2)
```

### 2. Доступные траки

Система учитывает траки в следующих статусах:
- `idle` — свободен, готов к работе
- `at_delivery` — заканчивает разгрузку (prebook)
- `at_pickup` — заканчивает погрузку (prebook)

### 3. Интерфейс выбора

После успешных переговоров игрок видит:

**Карточка выбранного трака:**
- Название трака и имя водителя
- Текущее местоположение
- HOS часов осталось
- Настроение водителя
- Deadhead расстояние (если > 0)
- Статус (если не idle)
- Кнопка "Изменить" (если есть другие траки)

**Возможность изменить выбор:**
- Клик на "Изменить" открывает список всех доступных траков
- Каждый трак показывает deadhead расстояние
- Выбранный трак отмечен галочкой ✓
- Клик на трак выбирает его и закрывает список

### 4. Prebook (резервирование)

Если выбран трак со статусом `at_delivery` или `at_pickup`:
- Груз назначается траку как "следующий"
- Трак автоматически начнёт движение после завершения текущей операции
- Показывается статус: "🔄 Заканчивает разгрузку — начнёт после"

### 5. Подтверждение

Кнопка "✅ Подтвердить":
- Назначает груз выбранному траку
- Вызывает `assignLoadToTruck()` из gameStore
- Закрывает окно переговоров
- Трак начинает движение к точке погрузки

## Технические детали

### Компоненты

**NegotiationChat.tsx** — основной компонент:
- Интегрированный выбор трака
- Автоматический расчёт лучшего варианта
- UI для смены трака
- Прямое назначение через `assignLoadToTruck()`

**LoadBoardPanel.tsx** — упрощён:
- Убран отдельный `AssignModal`
- Убран `pendingLoad` state
- Переговоры сразу назначают трак

### Функции

**selectBestTruck(trucks, load)** — выбор оптимального трака:
```typescript
function selectBestTruck(trucks: Truck[], load: LoadOffer): Truck | null
```

**calculateDeadhead(fromCity, toCity)** — расчёт расстояния:
```typescript
function calculateDeadhead(fromCity: string, toCity: string): number
```

## Преимущества новой системы

1. **Меньше кликов** — не нужно открывать отдельное окно выбора трака
2. **Умный выбор** — система предлагает оптимальный вариант автоматически
3. **Прозрачность** — игрок видит почему выбран именно этот трак (deadhead, HOS)
4. **Гибкость** — можно легко изменить выбор если нужно
5. **Prebook** — можно резервировать груз для занятого трака

## Что делать если нет свободных траков

Если все траки заняты:
- Кнопка "📞 Позвонить брокеру" становится неактивной
- Показывается сообщение: "🚫 Нет свободных траков"
- Игрок может использовать prebook для траков на разгрузке/погрузке
