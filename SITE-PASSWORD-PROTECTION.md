# 🔒 ПАРОЛЬНАЯ ЗАЩИТА САЙТА

## ✅ Что было сделано

Добавлена система парольной защиты для всего сайта в режиме разработки.

---

## 📁 Созданные файлы

### 1. `password-gate.html`
Красивая страница входа с полем для ввода пароля.

**Особенности:**
- Современный дизайн с градиентами
- Анимации и эффекты
- Адаптивная верстка (desktop + mobile)
- Сохранение доступа в localStorage

### 2. `site-protection.js`
Скрипт защиты, который проверяет доступ на всех страницах.

**Функции:**
- Автоматическая проверка при загрузке страницы
- Перенаправление на страницу входа при отсутствии доступа
- Работает для корневых страниц и папки `pages/`
- Защита от обхода через localStorage

### 3. `add-site-protection.ps1`
PowerShell скрипт для автоматического добавления защиты на все HTML страницы.

---

## 🚀 Как использовать

### Шаг 1: Добавить защиту на все страницы

Запустите PowerShell скрипт:

```powershell
.\add-site-protection.ps1
```

Скрипт автоматически:
- Добавит `<script src="site-protection.js">` на все страницы
- Пропустит уже защищенные файлы
- Покажет статистику обновлений

### Шаг 2: Проверить работу

1. Откройте любую страницу сайта (например, `index.html`)
2. Вас автоматически перенаправит на `password-gate.html`
3. Введите пароль: **dev2026**
4. После успешного входа вы получите доступ ко всему сайту

## 🛠️ Технические детали

### Как работает защита

1. При загрузке любой страницы запускается `site-protection.js`
2. Скрипт проверяет наличие ключа `site_access_granted` в localStorage
3. Если ключа нет → перенаправление на `password-gate.html`
4. После ввода правильного пароля → ключ сохраняется в localStorage
5. Пользователь получает доступ ко всем страницам

### Защищенные страницы

- ✅ Все страницы в корне (кроме `password-gate.html`)
- ✅ Все страницы в папке `pages/`
- ✅ Все модули обучения
- ✅ Административные страницы

### Публичные страницы (без защиты)

- `password-gate.html` - страница входа

## 🔓 Сброс доступа

### Для пользователей

Очистите localStorage в браузере:
1. Откройте DevTools (F12)
2. Перейдите в Application → Local Storage
3. Удалите ключ `site_access_granted`
4. Обновите страницу

### Для разработчиков

В консоли браузера выполните:

```javascript
resetSiteAccess()
```

## ⚠️ Важные замечания

1. **Это НЕ серьезная защита!** Пароль хранится в JavaScript и легко обходится.
2. Подходит только для защиты сайта в разработке от случайных посетителей.
3. Для продакшена используйте серверную авторизацию (PHP, Node.js, etc.)
