# Task 11.1 - Визуальное руководство по сообщениям об ошибках

## Обзор

Этот документ показывает все типы сообщений об ошибках Speech Recognition и когда они появляются.

---

## 1. 🎙️ Браузер не поддерживает голосовой ввод

### Когда появляется
- Браузер не поддерживает Web Speech API (например, Firefox)
- При загрузке страницы, если `window.SpeechRecognition` и `window.webkitSpeechRecognition` отсутствуют

### Сообщение
```
🎙️ Голосовой ввод не поддерживается в этом браузере. 
Используйте Chrome, Edge или Safari.
```

### Тип уведомления
⚠️ **Warning** (оранжевый фон)

### Дополнительные действия
- Кнопка микрофона отключается (`disabled=true`)
- Добавляется tooltip: "Голосовой ввод не поддерживается в этом браузере"

### Логирование
```javascript
console.error('[Speech Recognition] Not supported in this browser')
```

### Как протестировать
1. Откройте `pages/ai-broker-chat.html` в Firefox
2. Проверьте что кнопка микрофона неактивна
3. Наведите курсор на кнопку - должен появиться tooltip

---

## 2. 🎙️ Доступ к микрофону заблокирован

### Когда появляется
- Пользователь заблокировал доступ к микрофону в настройках браузера
- Ошибки: `not-allowed` или `permission-denied`

### Сообщение
```
🎙️ Доступ к микрофону заблокирован. 
Нажмите на иконку замка в адресной строке и разрешите доступ к микрофону.
```

### Тип уведомления
❌ **Error** (красный фон)

### Дополнительные действия
- Запись останавливается (`stopRec()`)
- Кнопка микрофона возвращается в нормальное состояние

### Логирование
```javascript
console.error('[Speech Recognition] Microphone permission denied')
```

### Как протестировать
1. Откройте `pages/ai-broker-chat.html` в Chrome
2. Нажмите на иконку замка в адресной строке
3. Заблокируйте доступ к микрофону
4. Попробуйте нажать на кнопку микрофона
5. Должно появиться сообщение с инструкцией

### Инструкция для пользователя
1. Нажмите на иконку замка 🔒 слева от адресной строки
2. Найдите "Микрофон"
3. Выберите "Разрешить"
4. Обновите страницу

---

## 3. 🎙️ Микрофон не найден

### Когда появляется
- Микрофон физически не подключен
- Микрофон отключен в системных настройках
- Ошибка: `audio-capture`

### Сообщение
```
🎙️ Микрофон не найден. 
Проверьте подключение микрофона и попробуйте снова.
```

### Тип уведомления
❌ **Error** (красный фон)

### Дополнительные действия
- Запись останавливается (`stopRec()`)

### Логирование
```javascript
console.error('[Speech Recognition] Audio capture failed - no microphone')
```

### Как протестировать
1. Отключите микрофон в системных настройках
2. Откройте `pages/ai-broker-chat.html`
3. Попробуйте начать запись
4. Должно появиться сообщение

### Что проверить
- Микрофон подключен к компьютеру
- Микрофон включен в системных настройках
- Микрофон не используется другим приложением

### Сообщение
```
🌐 Ошибка сети. 
Проверьте подключение к интернету и попробуйте снова.
```

### Тип уведомления
❌ **Error** (красный фон)

### Дополнительные действия
- Запись останавливается (`stopRec()`)

### Логирование
```javascript
console.error('[Speech Recognition] Network error')
```

### Как протестировать
1. Отключите интернет
2. Попробуйте использовать голосовой ввод
3. Должно появиться сообщение

### Примечание
Web Speech API требует подключения к интернету, так как распознавание происходит на серверах Google.

## 5. 🔒 Требуется HTTPS соединение

### Когда появляется

- Ошибка: `service-not-allowed`

### Сообщение
```
🔒 Голосовой ввод требует HTTPS соединение. 
Используйте безопасное подключение.
```

### Тип уведомления
❌ **Error** (красный фон)

### Дополнительные действия
- Запись останавливается (`stopRec()`)

### Логирование
```javascript
console.error('[Speech Recognition] Service not allowed - check HTTPS')
```

### Как протестировать
1. Откройте страницу по HTTP (не HTTPS)
2. Попробуйте использовать голосовой ввод
3. Должно появиться сообщение

### Решение
- Используйте HTTPS для production
- Для локальной разработки используйте `localhost` (разрешен без HTTPS)

## 7. 🎙️ Голосовой ввод недоступен (не инициализирован)

### Когда появляется
- Попытка запустить запись когда `recognition` не инициализирован
- Внутренняя ошибка инициализации

### Сообщение
```
🎙️ Голосовой ввод недоступен. 
Обновите страницу или используйте другой браузер.
```

### Тип уведомления
❌ **Error** (красный фон)

### Логирование
```javascript
console.error('[Speech Recognition] Recognition not initialized')
```

### Решение
- Обновить страницу (F5)
- Попробовать другой браузер

---

## 8. ❌ Не удалось запустить распознавание

### Когда появляется
- Ошибка при вызове `recognition.start()`
- Исключая `InvalidStateError` (уже запущено)

### Сообщение
```
❌ Не удалось запустить распознавание речи. 
Попробуйте снова.
```

### Тип уведомления
❌ **Error** (красный фон)

### Логирование
```javascript
console.error('[Speech Recognition] Failed to start:', e)
```

## Специальные случаи (без уведомлений)

### 1. no-speech (Нет речи)

**Когда происходит:**
- Пользователь молчит во время записи
- Естественная пауза в речи

**Поведение:**
- ❌ Уведомление НЕ показывается
- ✅ Запись продолжается
- ✅ Логируется как `console.log` (не ошибка)

**Логирование:**
```javascript
console.log('[Speech Recognition] No speech detected (normal pause)')
```

**Почему без уведомления:**
Это нормальное поведение - пользователь может делать паузы во время разговора. Показывать ошибку было бы раздражающим.

---

### 2. aborted (Прервано)

**Когда происходит:**
- Распознавание прервано пользователем
- Распознавание прервано системой

**Поведение:**
- ❌ Уведомление НЕ показывается
- ✅ Запись останавливается
- ✅ Логируется как `console.log`

**Логирование:**
```javascript
console.log('[Speech Recognition] Recognition aborted')
```

---

### 3. InvalidStateError (Уже запущено)

**Когда происходит:**
- Попытка запустить уже запущенное распознавание

**Поведение:**
- ❌ Уведомление НЕ показывается
- ✅ Запись продолжается
- ✅ Логируется как `console.log`

**Логирование:**
```javascript
console.log('[Speech Recognition] Already started, continuing...')
```

---

## Дополнительные уведомления

### file:// Protocol Warning

**Когда появляется:**
- Страница открыта локально через `file://` протокол

**Сообщение:**
```
⚠️ Страница открыта локально (file://)
Браузер не сохраняет разрешение микрофона.

Для нормальной работы запустите локальный сервер:
python -m http.server 8000
или используйте Live Server в VS Code
```

**Тип:**
⚠️ **Warning** (оранжевый фон)

**Примечание:**
Это уведомление НЕ автоматически исчезает - оно важное и должно оставаться видимым.

---

### Microphone Permission Tip

**Когда появляется:**
- При первой загрузке страницы
- Если разрешение микрофона в статусе `prompt` (еще не запрошено)
- Только если НЕ `file://` протокол

**Сообщение:**
```
🎙️ Нажми на микрофон и выбери "Разрешить когда я на сайте" — 
больше не будет спрашивать
```

**Тип:**
ℹ️ **Info** (синий фон)

**Поведение:**
- Автоматически исчезает через 7 секунд
- Показывается только один раз

## Цветовая схема уведомлений

### ❌ Error (Красный)
- **Фон:** `rgba(239, 68, 68, 0.15)`
- **Граница:** `rgba(239, 68, 68, 0.3)`
- **Использование:** Критические ошибки, требующие действий пользователя

### ⚠️ Warning (Оранжевый)
- **Фон:** `rgba(249, 115, 22, 0.15)`
- **Граница:** `rgba(249, 115, 22, 0.3)`
- **Использование:** Предупреждения, не блокирующие работу

### ℹ️ Info (Синий)
- **Фон:** `rgba(6, 182, 212, 0.15)`
- **Граница:** `rgba(6, 182, 212, 0.3)`
- **Использование:** Информационные сообщения, подсказки

## Рекомендации для пользователей

### Если голосовой ввод не работает:

1. **Проверьте браузер**
   - Используйте Chrome, Edge или Safari
   - Обновите браузер до последней версии

2. **Проверьте разрешения**
   - Нажмите на замок 🔒 в адресной строке
   - Разрешите доступ к микрофону

3. **Проверьте микрофон**
   - Микрофон подключен и включен
   - Микрофон работает в других приложениях
   - Микрофон выбран по умолчанию в системе

4. **Проверьте соединение**
   - Есть подключение к интернету
   - Страница открыта по HTTPS

5. **Попробуйте обновить страницу**
   - Нажмите F5 или Ctrl+R
   - Очистите кэш браузера

---

## Заключение

Все ошибки Speech Recognition обрабатываются с понятными сообщениями для пользователей и детальным логированием для разработчиков. Система gracefully degradation обеспечивает работу даже при отсутствии поддержки голосового ввода.
