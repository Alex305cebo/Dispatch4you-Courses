# План реализации: Редизайн карточек в квиз-формат

## Обзор

Замена механики флип-карточек на квиз-формат с 3 вариантами ответа в файле `pages/Trainer-Quiz.html`. Реализация QuizEngine, DistractorPicker и FeedbackUI как JavaScript-модулей внутри существующей страницы. Сохранение SRS-системы и интеграции с `xp-system.js`.

- [x] 2. Реализовать QuizEngine — управление сессией и SRS
  - [x] 2.1 Создать объект QuizSession и функцию `initQuizSession(cards)`
    - Инициализирует сессию: cards, currentIndex=0, correctCount=0, wrongCount=0, streak=0, maxStreak=0
    - Фильтрация карточек по категории и сложности (используя существующие фильтры)
    - SRS-сортировка: карточки с просроченным `next` идут первыми
    - _Требования: 5.1, 5.2, 5.3_

  - [x] 2.2 Создать функцию `submitAnswer(session, selectedCardId, currentCard)`

    - При неправильном ответе: reps = 0, next = now + 1 день, streak = 0
    - При правильном ответе: вызвать `awardXP('QUIZ_CORRECT')`
    - Сохранить обновлённый TrainerState в localStorage
    - _Требования: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6_

  - [x] 2.3 Создать функцию `nextQuestion(session)` для перехода к следующему вопросу

    - Если все карточки пройдены — возвращает null (сигнал для экрана результатов)
    - _Требования: 7.2, 7.3_

  - [ ]* 2.4 Написать property-тест для SRS при правильном ответе
    - **Property 3: Корректность SRS-обновления при правильном ответе**
    - Для любого reps ≥ 0: reps' = reps + 1, next' = now + SRS_INTERVALS[min(reps, 4)]

  - [ ]* 2.5 Написать property-тест для SRS при неправильном ответе

    - **Проверяет: Требование 4.2**

  - [ ]* 2.6 Написать property-тест для streak
