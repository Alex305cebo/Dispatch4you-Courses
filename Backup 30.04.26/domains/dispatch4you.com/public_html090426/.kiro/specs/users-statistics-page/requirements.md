# Requirements Document

## Введение

Страница статистики всех пользователей (pages/users-stats.html) — административная/публичная страница обучающей платформы для диспетчеров грузоперевозок. Страница отображает детальную статистику по каждому студенту: XP, уровень, прогресс по курсу (15 уроков), модулям (12 тестов), кейсам (50 кейсов), карточкам (80 карточек), streak, точность ответов, время на платформе и достижения. Включает общую аналитику (overview), лидерборд, фильтрацию, сортировку, детальный профиль пользователя и экспорт данных в CSV. Страница использует тёмную тему проекта, Firebase Firestore для данных, адаптивный дизайн для десктопа и мобильных устройств.

## Глоссарий

- **Stats_Page**: Веб-страница статистики всех пользователей (pages/users-stats.html)
- **Overview_Panel**: Панель общей статистики платформы (всего пользователей, активных, средний XP, средний прогресс)
- **User_Card**: Карточка пользователя в списке с основной информацией и мини-прогрессом
- **User_Profile_Modal**: Модальное окно с детальной статистикой конкретного пользователя
- **Leaderboard**: Таблица рейтинга пользователей с сортировкой по XP, streak или прогрессу
- **Activity_Chart**: График активности пользователей по дням/неделям
- **Progress_Bar**: Визуальный индикатор прогресса (полоска с процентом заполнения)
- **Mini_Chart**: Компактный график прогресса на карточке пользователя
- **Search_Filter**: Компонент поиска и фильтрации пользователей по имени, email, уровню
- **Sort_Control**: Компонент сортировки списка пользователей по различным критериям
- **CSV_Exporter**: Модуль экспорта данных пользователей в CSV-файл
- **Student**: Пользователь платформы, зарегистрированный в Firebase Authentication и имеющий документ в коллекции Firestore `users`
- **Superuser**: Администратор с ролью superuser, имеющий полный доступ ко всем страницам
- **Firestore_Users**: Коллекция `users` в Firebase Firestore, содержащая данные пользователей (xp, stats, loginStreak, trainerStats, role, lastActivity и др.)

## Требования

### Requirement 1: Доступ и авторизация

**User Story:** Как администратор, я хочу, чтобы страница статистики была доступна только авторизованным пользователям с правами superuser, чтобы защитить персональные данные студентов.

#### Acceptance Criteria

1. WHEN неавторизованный пользователь открывает Stats_Page, THE System SHALL перенаправить пользователя на страницу входа
2. WHEN авторизованный пользователь с ролью, отличной от superuser, открывает Stats_Page, THE System SHALL отобразить сообщение об отсутствии доступа
3. THE Stats_Page SHALL использовать существующий role-guard.js для проверки авторизации и роли
4. THE Stats_Page SHALL использовать существующую систему Firebase Authentication для определения текущего пользователя

### Requirement 2: Загрузка данных пользователей из Firestore

**User Story:** Как администратор, я хочу видеть актуальные данные всех пользователей, чтобы отслеживать прогресс обучения на платформе.

#### Acceptance Criteria

1. WHEN Stats_Page загружается, THE Stats_Page SHALL запросить все документы из коллекции Firestore_Users
2. THE Stats_Page SHALL извлечь из каждого документа пользователя следующие поля: displayName, email, photoURL, xp, role, stats, loginStreak, lastActivity, trainerStats, createdAt
3. WHEN данные загружаются, THE Stats_Page SHALL отобразить индикатор загрузки (скелетон или спиннер)
4. IF запрос к Firestore завершается ошибкой, THEN THE Stats_Page SHALL отобразить сообщение об ошибке с кнопкой повторной попытки
5. WHEN данные загружены, THE Stats_Page SHALL рассчитать производные метрики для каждого пользователя: уровень (getLevelByXP), точность ответов (correctAnswers/totalAnswers), прогресс по курсу, модулям, кейсам и карточкам

### Requirement 3: Панель общей статистики (Overview)

**User Story:** Как администратор, я хочу видеть сводную статистику по всей платформе, чтобы быстро оценить общее состояние обучения.

#### Acceptance Criteria

1. THE Overview_Panel SHALL отображать общее количество зарегистрированных пользователей
2. THE Overview_Panel SHALL отображать количество активных пользователей за последние 7 дней (на основе поля lastActivity)
3. THE Overview_Panel SHALL отображать количество активных пользователей за последние 30 дней
4. THE Overview_Panel SHALL отображать средний XP по всем пользователям
5. THE Overview_Panel SHALL отображать средний прогресс по курсу (процент пройденных уроков из 15)
6. THE Overview_Panel SHALL отображать средний streak по всем пользователям
7. THE Overview_Panel SHALL использовать карточки статистики с иконками и цветовой индикацией, согласно стилю admin.html

### Requirement 4: График активности пользователей

**User Story:** Как администратор, я хочу видеть график активности пользователей по дням, чтобы отслеживать тренды вовлечённости.

#### Acceptance Criteria

1. THE Activity_Chart SHALL отображать количество активных пользователей по дням за последние 30 дней
2. THE Activity_Chart SHALL использовать библиотеку Chart.js для рендеринга графика
3. THE Activity_Chart SHALL предоставить переключатель периода: 7 дней, 30 дней
4. THE Activity_Chart SHALL использовать цветовую схему проекта (--primary, --secondary) для линий и заливки графика
5. THE Activity_Chart SHALL отображать tooltip с точным значением при наведении на точку графика

### Requirement 5: Лидерборд (Leaderboard)

**User Story:** Как администратор, я хочу видеть рейтинг студентов по различным критериям, чтобы выявлять лучших и отстающих.

#### Acceptance Criteria

1. THE Leaderboard SHALL отображать топ-10 пользователей по XP с указанием ранга, имени, уровня и XP
2. THE Leaderboard SHALL предоставить переключатель критерия рейтинга: по XP, по streak, по общему прогрессу
3. WHEN критерий рейтинга изменяется, THE Leaderboard SHALL пересортировать и обновить список пользователей
4. THE Leaderboard SHALL выделять первые три позиции медалями (🥇, 🥈, 🥉)
5. THE Leaderboard SHALL отображать уровень пользователя с цветным бейджем (role-badge), согласно стилю admin.html

### Requirement 6: Список пользователей (User Cards)

**User Story:** Как администратор, я хочу видеть список всех пользователей с основной информацией, чтобы быстро оценить статус каждого студента.

#### Acceptance Criteria

1. THE Stats_Page SHALL отображать список всех пользователей в виде карточек (User_Card)
2. THE User_Card SHALL содержать: аватар (photoURL или инициалы), имя, email, дату регистрации
3. THE User_Card SHALL содержать: текущий уровень и XP с цветным бейджем уровня
4. THE User_Card SHALL содержать: Progress_Bar прогресса по курсу (N из 15 уроков)
5. THE User_Card SHALL содержать: Progress_Bar прогресса по модулям (N из 12 тестов)
6. THE User_Card SHALL содержать: Progress_Bar прогресса по кейсам (N из 50 кейсов)
7. THE User_Card SHALL содержать: Progress_Bar прогресса по карточкам (N из 80 карточек)
8. THE User_Card SHALL содержать: streak (дни подряд) и дату последней активности
9. THE User_Card SHALL содержать: точность ответов в процентах
10. WHEN пользователь нажимает на User_Card, THE Stats_Page SHALL открыть User_Profile_Modal с детальной информацией

### Requirement 7: Поиск и фильтрация пользователей

**User Story:** Как администратор, я хочу искать и фильтровать пользователей, чтобы быстро находить нужного студента.

#### Acceptance Criteria

1. THE Search_Filter SHALL предоставить текстовое поле для поиска по имени или email пользователя
2. WHEN администратор вводит текст в поле поиска, THE Stats_Page SHALL фильтровать список пользователей в реальном времени (по мере ввода)
3. THE Search_Filter SHALL предоставить фильтр по уровню пользователя (выпадающий список уровней из LEVELS)
4. THE Search_Filter SHALL предоставить фильтр по статусу активности: все, активные за 7 дней, активные за 30 дней, неактивные
5. WHEN фильтры применяются, THE Stats_Page SHALL обновить список пользователей и счётчик результатов

### Requirement 8: Сортировка пользователей

**User Story:** Как администратор, я хочу сортировать пользователей по различным критериям, чтобы анализировать данные в нужном порядке.

#### Acceptance Criteria

1. THE Sort_Control SHALL предоставить сортировку по XP (по убыванию и возрастанию)
2. THE Sort_Control SHALL предоставить сортировку по дате последней активности
3. THE Sort_Control SHALL предоставить сортировку по общему прогрессу (среднее по всем категориям)
4. THE Sort_Control SHALL предоставить сортировку по дате регистрации
5. THE Sort_Control SHALL предоставить сортировку по streak
6. WHEN критерий сортировки изменяется, THE Stats_Page SHALL пересортировать и обновить список пользователей

### Requirement 9: Детальный профиль пользователя (модальное окно)

**User Story:** Как администратор, я хочу видеть детальную статистику конкретного пользователя, чтобы оценить его прогресс и вовлечённость.

#### Acceptance Criteria

1. WHEN администратор нажимает на User_Card, THE User_Profile_Modal SHALL открыться с анимацией
2. THE User_Profile_Modal SHALL отображать аватар, полное имя, email и дату регистрации пользователя
3. THE User_Profile_Modal SHALL отображать текущий уровень, XP и прогресс до следующего уровня (Progress_Bar)
4. THE User_Profile_Modal SHALL отображать детальный прогресс по курсу: список 15 уроков с отметкой пройденных
5. THE User_Profile_Modal SHALL отображать детальный прогресс по модулям: список 12 тестов с отметкой пройденных
6. THE User_Profile_Modal SHALL отображать количество изученных кейсов из 50 и карточек из 80
7. THE User_Profile_Modal SHALL отображать streak (текущий и максимальный), точность ответов (%), общее количество ответов
8. THE User_Profile_Modal SHALL отображать дату и время последней активности
9. THE User_Profile_Modal SHALL отображать Mini_Chart с динамикой XP пользователя (на основе xpHistory)
10. WHEN администратор нажимает кнопку закрытия или область вне модального окна, THE User_Profile_Modal SHALL закрыться с анимацией

### Requirement 10: Экспорт данных в CSV

**User Story:** Как администратор, я хочу экспортировать данные пользователей в CSV, чтобы анализировать статистику во внешних инструментах.

#### Acceptance Criteria

1. THE Stats_Page SHALL предоставить кнопку "Экспорт CSV" в панели управления
2. WHEN администратор нажимает кнопку экспорта, THE CSV_Exporter SHALL сформировать CSV-файл с данными всех отображаемых (отфильтрованных) пользователей
3. THE CSV_Exporter SHALL включить в файл столбцы: имя, email, уровень, XP, прогресс по курсу (%), прогресс по модулям (%), прогресс по кейсам (%), прогресс по карточкам (%), streak, точность (%), дата последней активности, дата регистрации
4. THE CSV_Exporter SHALL использовать кодировку UTF-8 с BOM для корректного отображения кириллицы в Excel
5. THE CSV_Exporter SHALL инициировать скачивание файла с именем формата `users-stats-YYYY-MM-DD.csv`

### Requirement 11: Адаптивный дизайн

**User Story:** Как администратор, я хочу использовать страницу статистики на любом устройстве, чтобы проверять данные с телефона или планшета.

#### Acceptance Criteria

1. THE Stats_Page SHALL корректно отображаться на экранах шириной от 320px до 1920px
2. THE Stats_Page SHALL адаптировать grid карточек Overview_Panel: 4 колонки на десктопе (>1024px), 2 колонки на планшете (768px-1024px), 1 колонка на мобильном (<768px)
3. THE Stats_Page SHALL адаптировать grid User_Card: 3 колонки на десктопе, 2 на планшете, 1 на мобильном
4. THE Stats_Page SHALL адаптировать Activity_Chart для мобильных устройств (уменьшение высоты, горизонтальный скролл при необходимости)
5. THE User_Profile_Modal SHALL занимать полный экран на мобильных устройствах (<768px)
6. THE Stats_Page SHALL реализовать мобильное гамбургер-меню для навигации на экранах <768px
7. THE Stats_Page SHALL обеспечить минимальный размер touch-target 44x44px для всех интерактивных элементов на мобильных устройствах

### Requirement 12: Стилизация и интеграция с проектом

**User Story:** Как администратор, я хочу, чтобы страница статистики выглядела как часть платформы, чтобы обеспечить единый пользовательский опыт.

#### Acceptance Criteria

1. THE Stats_Page SHALL быть доступна по адресу pages/users-stats.html
2. THE Stats_Page SHALL использовать тёмную тему проекта с CSS-переменными (:root), согласно стилю admin.html и dashboard.html
3. THE Stats_Page SHALL интегрироваться с существующей навигацией сайта (shared-nav.css, nav-loader.js)
4. THE Stats_Page SHALL использовать шрифт Inter, согласно стилю проекта
5. THE Stats_Page SHALL использовать существующие CSS-классы для бейджей ролей (role-1 ... role-10) из admin.html
6. THE Stats_Page SHALL использовать библиотеку Chart.js для графиков (подключение через CDN)
7. THE Stats_Page SHALL использовать Font Awesome для иконок (подключение через CDN)
8. THE Stats_Page SHALL включать hero-секцию с заголовком "Статистика пользователей" и кратким описанием
