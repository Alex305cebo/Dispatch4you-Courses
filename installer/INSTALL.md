# Dispatch4you — расширение для DAT: деплой и установка

Расширение ставится диспетчерам **одним кликом** через политику Chrome `ExtensionInstallForcelist`,
которая указывает на самостоятельно размещённый `.crx`. Ключ Groq **не зашит в расширении** —
он лежит на сервере, а расширение ходит к нему через PHP-посредник `api/groq.php`.

Постоянный ID расширения: `ahbjapbcfgplnnjempddicmohpbdloin`
Деплой сайта: `git push` в `main` → GitHub Actions (`.github/workflows/deploy.yml`) → Hostinger.

---

## ⚠️ Шаг 0 — один раз настроить ключ на сервере (ОБЯЗАТЕЛЬНО)
Без этого диктовка и перевод работать не будут.

1. Зайти в **Hostinger → Диспетчер файлов** сайта dispatch4you.com.
2. Подняться **на один уровень ВЫШЕ** папки `public_html` (т.е. в `domains/dispatch4you.com/`).
3. Создать там файл `groq.key`, внутри одной строкой — твой ключ Groq (вид `gsk_...`).
4. Сохранить. Этот файл **не в интернете и не в git** — его видит только PHP на сервере.

> Прежний ключ был «засвечен» (лежал прямо в коде расширения). Зайди в личный кабинет Groq,
> **выпусти новый ключ**, положи его в `groq.key`, а старый **отзови/удали**. Так старый ключ,
> который уже мог утечь, перестанет работать.

## Шаг 1 — выложить расширение на сайт (деплой)
Файлы `ext/update.xml`, `ext/dispatch4you.crx` и `api/groq.php` уже в репозитории.
Просто запушить:
```
git push
```
GitHub Actions сам зальёт их на сайт. Проверить, что открываются:
- https://dispatch4you.com/ext/update.xml
- https://dispatch4you.com/ext/dispatch4you.crx

## Шаг 2 — дать диспетчерам установщик
Раздать файл `installer/install-dispatch4you.bat` (двойной клик → подтвердить запрос прав
администратора UAC — иначе Windows не даёт писать политику Chrome).
Либо собрать красивый `.exe` из `installer/installer.iss` в Inno Setup (делает то же самое).
Диспетчер запускает → перезапускает Chrome → расширение работает на one.dat.com.

---

## Обновление расширения (потом)
1. Поднять `"version"` в `DispatchPro extension/manifest.json` (напр. `1.0.1`).
2. Тем же числом — `version` в `ext/update.xml`.
3. Переупаковать:
   ```
   powershell -ExecutionPolicy Bypass -File build\pack.ps1
   ```
   (обновит `ext/dispatch4you.crx` тем же ключом → ID не меняется).
4. `git push`.

Chrome у всех диспетчеров обновится сам в течение нескольких часов. `.exe`/`.bat` перевыпускать не нужно.

## Файлы
- `DispatchPro extension/` — само расширение; в `manifest.json` поле `"key"` фиксирует ID.
- `api/groq.php` — посредник к Groq (деплоится на сайт; ключ читает из `groq.key` выше веб-корня).
- `ext/update.xml`, `ext/dispatch4you.crx` — то, что Chrome скачивает (деплоится на сайт).
- `build/key.pem` — **СЕКРЕТНЫЙ** ключ подписи. Локально, в `.gitignore`. Не терять, не пересоздавать,
  держать офлайн-копию (потеряешь — ID сменится и обновления сломаются).
- `build/pack.ps1` — упаковщик (локально).
- `installer/install-dispatch4you.bat` / `uninstall-dispatch4you.bat` — для диспетчеров.
- `installer/installer.iss` — сборка `.exe` (Inno Setup).

## Проверка
- `chrome://policy` → есть `ExtensionInstallForcelist` с нашим ID и URL.
- `chrome://extensions` → расширение есть, включено, «Установлено корпоративной политикой».
- На https://one.dat.com появляются синяя панель / кнопки / диктовка.
- Диктовка/перевод работают → значит `groq.key` на сервере настроен верно.

## Важно знать
- Force-install показывает баннер «Управляется вашей организацией» и не даёт пользователю
  удалить расширение вручную — это нормально для такого способа.
- Установка пишется в `HKLM` и **требует прав администратора** (UAC). Так сделано потому, что
  Windows защищает ветку политик Chrome — обычный пользователь в неё писать не может (была ошибка
  «Access is denied» при попытке per-user). `.bat` сам запрашивает повышение прав.
- Посредник `api/groq.php` использует твой ключ; он проверяет, что запрос идёт со страницы DAT
  (Origin/Referer). Это останавливает случайное злоупотребление, но не «упёртого» атакующего —
  при росте трафика стоит добавить токен на диспетчера. Ключ всегда можно мгновенно сменить
  в `groq.key`, не трогая расширение.
