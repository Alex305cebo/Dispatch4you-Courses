# Task 11.1 - Speech Recognition Error Handling ✅

## Описание задачи

Реализована комплексная обработка ошибок Speech Recognition API с понятными сообщениями для пользователей и детальным логированием для отладки.

## Выполненные изменения

### 1. Проверка поддержки Speech Recognition

**Файл:** `pages/ai-broker-chat.html`

**Изменения:**
- Добавлена проверка наличия Speech Recognition API при загрузке страницы
- Если API не поддерживается:
  - Кнопка микрофона отключается (`disabled=true`)
  - Добавляется tooltip с объяснением
  - Показывается уведомление с рекомендацией использовать Chrome, Edge или Safari
  - Ошибка логируется в `console.error`

```javascript
// Check if Speech Recognition is supported
if(!SR){

```

### 2. Обработка различных типов ошибок

**Реализован switch-case для обработки всех типов ошибок Speech Recognition:**

#### a) `no-speech` - Тихая обработка
- **Поведение:** Не показывается уведомление (нормальная пауза пользователя)
- **Логирование:** `console.log` (не ошибка)
- **Действие:** Продолжение работы без остановки записи

```javascript
case 'no-speech':
  // Silent handling - user just paused, don't show error
  console.log('[Speech Recognition] No speech detected (normal pause)');
  break;
```

#### b) `not-allowed` / `permission-denied` - Блокировка микрофона
- **Поведение:** Показывается инструкция по разрешению доступа
- **Уведомление:** "🎙️ Доступ к микрофону заблокирован. Нажмите на иконку замка в адресной строке и разрешите доступ к микрофону."
- **Логирование:** `console.error`
- **Действие:** Остановка записи

```javascript
case 'not-allowed':
case 'permission-denied':
  console.error('[Speech Recognition] Microphone permission denied');
  stopRec();
  showNotification('🎙️ Доступ к микрофону заблокирован. Нажмите на иконку замка в адресной строке и разрешите доступ к микрофону.','error');
  break;
```

#### c) `audio-capture` - Отсутствие микрофона
- **Поведение:** Показывается сообщение о проблеме с микрофоном
- **Уведомление:** "🎙️ Микрофон не найден. Проверьте подключение микрофона и попробуйте снова."
- **Логирование:** `console.error`
- **Действие:** Остановка записи

```javascript
case 'audio-capture':
  console.error('[Speech Recognition] Audio capture failed - no microphone');
  stopRec();
  showNotification('🎙️ Микрофон не найден. Проверьте подключение микрофона и попробуйте снова.','error');
  break;
```

#### d) `network` - Сетевая ошибка
- **Поведение:** Показывается сообщение о проблеме с интернетом
- **Уведомление:** "🌐 Ошибка сети. Проверьте подключение к интернету и попробуйте снова."
- **Логирование:** `console.error`
- **Действие:** Остановка записи

```javascript
case 'network':
  console.error('[Speech Recognition] Network error');
  stopRec();
  showNotification('🌐 Ошибка сети. Проверьте подключение к интернету и попробуйте снова.','error');
  break;
```

#### e) `aborted` - Прерывание распознавания
- **Поведение:** Тихая обработка (обычно вызвано пользователем или системой)
- **Логирование:** `console.log`
- **Действие:** Остановка записи

```javascript
case 'aborted':
  console.log('[Speech Recognition] Recognition aborted');
  stopRec();
  break;
```

#### f) `service-not-allowed` - Проблема с HTTPS
- **Поведение:** Показывается сообщение о необходимости HTTPS
- **Уведомление:** "🔒 Голосовой ввод требует HTTPS соединение. Используйте безопасное подключение."
- **Логирование:** `console.error`
- **Действие:** Остановка записи

```javascript
case 'service-not-allowed':
  console.error('[Speech Recognition] Service not allowed - check HTTPS');
  stopRec();
  showNotification('🔒 Голосовой ввод требует HTTPS соединение. Используйте безопасное подключение.','error');
  break;
```

#### g) Неизвестные ошибки
- **Поведение:** Показывается общее сообщение об ошибке
- **Уведомление:** "❌ Ошибка распознавания речи. Попробуйте снова."
- **Логирование:** `console.error` с деталями ошибки
- **Действие:** Остановка записи

```javascript
default:
  console.error('[Speech Recognition] Unknown error:', e.error);
  stopRec();
  showNotification('❌ Ошибка распознавания речи. Попробуйте снова.','error');
```

### 3. Улучшенная функция startRec()

**Добавлена проверка инициализации и обработка ошибок запуска:**

```javascript
function startRec(){
  if(!recognition){
    console.error('[Speech Recognition] Recognition not initialized');
    showNotification('🎙️ Голосовой ввод недоступен. Обновите страницу или используйте другой браузер.','error');
    return;
  }
  if(busy)return;
  
  isRec=true;
  finalText='';
  TranscriptDisplay.clear();
  micBtn.classList.add('rec');
  
  try{
    recognition.start();
  }catch(e){
    console.error('[Speech Recognition] Failed to start:', e);
    // If already started, this is normal - ignore
    if(e.name==='InvalidStateError'){
      console.log('[Speech Recognition] Already started, continuing...');
    }else{
      // Other errors - show notification
      stopRec();
      showNotification('❌ Не удалось запустить распознавание речи. Попробуйте снова.','error');
    }
  }
}
```

### 4. Логирование всех ошибок

**Все ошибки логируются с префиксом `[Speech Recognition]` для удобной фильтрации:**

```javascript
recognition.onerror=e=>{
  console.error('[Speech Recognition] Error:', e.error, e);
  // ... обработка ошибок
};
```

## Соответствие требованиям

### ✅ Requirement 11.1
**"WHEN Speech_Recognition недоступен в браузере, THE System SHALL отобразить сообщение с предложением использовать текстовый режим"**

**Реализация:** При отсутствии поддержки показывается уведомление с рекомендацией использовать Chrome, Edge или Safari. Кнопка микрофона отключается.

### ✅ Requirement 11.3
**"WHEN микрофон заблокирован браузером, THE System SHALL отобразить инструкцию по разрешению доступа"**

**Реализация:** При ошибках `not-allowed` или `permission-denied` показывается детальная инструкция: "Нажмите на иконку замка в адресной строке и разрешите доступ к микрофону."

### ✅ Requirement 11.5
**"THE System SHALL логировать все ошибки в консоль для отладки"**

**Реализация:** Все ошибки логируются через `console.error` с префиксом `[Speech Recognition]` и полными деталями ошибки. Нормальные события (no-speech, aborted) логируются через `console.log`.

## Технические детали

### Обработка autoSendTimer

При любой ошибке очищается таймер автоматической отправки:

```javascript
if(autoSendTimer){
  clearTimeout(autoSendTimer);
  autoSendTimer=null;
}
```

### Обработка InvalidStateError

При попытке запустить уже запущенное распознавание обрабатывается тихо:

```javascript
if(e.name==='InvalidStateError'){
  console.log('[Speech Recognition] Already started, continuing...');
}
```

### Использование существующей системы уведомлений

Все уведомления используют существующую функцию `showNotification()` с типами:
- `'error'` - красный фон для критических ошибок
- `'warning'` - оранжевый фон для предупреждений
- `'info'` - синий фон для информационных сообщений
