# 🎓 Tutorial System — Краткое описание

## Что это?

Модульная система **нарративного обучения** для DispatcherTraining. Игрок учится через контекстные задачи от персонажа-наставника Майка, а не через статичные туториалы.

---

## 🏗️ Архитектура

### 4 изолированных модуля:

1. **TutorialStateMachine.js** — State Machine (FSM)
   - Отслеживает текущий шаг обучения
   - Контролирует переходы между шагами
   - Валидирует действия игрока

2. **MentorshipData.js** — Data Layer
   - Хранит диалоги наставника
   - Определяет задачи и подсказки
   - Легко редактируется без изменения кода

3. **TutorialUIController.js** — UI Controller
   - Показывает диалоги наставника
   - Подсвечивает элементы на странице
   - Управляет анимациями и tooltip

4. **TutorialManager.js** — Главный оркестратор
   - Связывает все модули
   - Управляет жизненным циклом обучения
   - Интегрируется с игровой логикой

---

## 🎯 Ключевые фичи

### ✅ Модульность
Каждый модуль изолирован и отвечает за одну задачу. Можно менять один модуль не трогая другие.

### ✅ Масштабируемость
Добавление нового шага = добавление одного объекта в `MentorshipData.js`. Никакого переписывания логики.

### ✅ Интеграция с существующим UI
Работает с твоей 2-колоночной вёрсткой документов. Не ломает существующий код.

### ✅ Event-Driven
Использует Observer pattern и Event Bus для связи между модулями.

### ✅ Сохранение прогресса
Автоматически сохраняет прогресс в localStorage. Игрок может продолжить с места остановки.

## 🎮 Как это работает

### Пример: Шаг "Проверка Rate Confirmation"

1. **State Machine** переключается на `STEP_4_RATE_CON_REVIEW`
2. **UI Controller** показывает диалог наставника:
   ```
   👨‍💼 Майк · Старший диспетчер
   
   Rate Confirmation — это твой контракт с брокером.
   Проверь все ключевые поля.
   
   📋 Твоя задача: Проверь все поля в Rate Confirmation
   ```
3. **UI Controller** подсвечивает документ на странице
4. Игрок кликает на поля документа
5. **Manager** отслеживает клики через контекст
6. Когда игрок проверил 5+ полей → переход к следующему шагу

## 📊 Структура шага обучения

```javascript

  title: 'Проверка Rate Confirmation',
  mentorDialog: 'Текст наставника...',
  task: 'Задача для игрока',
  ui: {
    highlight: '#doc-ratecon',
    position: 'left',
    pulse: true
  },
  validation: {
    type: 'custom',
    check: (context) => context.checkedFields.size >= 5
  },
  nextSteps: ['STEP_5_RATE_CON_ACCEPT']
}
```

## 🎯 11 шагов обучения

1. **STEP_1_WELCOME** — Приветствие в офисе
2. **STEP_2_MORNING_BRIEFING** — Проверка флота траков
3. **STEP_3_INBOX_CHECK** — Проверка входящей почты
4. **STEP_4_RATE_CON_REVIEW** — Проверка Rate Confirmation
5. **STEP_5_RATE_CON_ACCEPT** — Принятие груза
6. **STEP_6_DRIVER_DISPATCH** — Отправка задания водителю
7. **STEP_7_MONITOR_PROGRESS** — Мониторинг прогресса
8. **STEP_8_DRIVER_QUESTION** — Вопрос от водителя
9. **STEP_9_PROBLEM_SOLVED** — Решение проблемы
10. **STEP_10_DELIVERY_COMPLETE** — Доставка завершена
11. **STEP_11_TUTORIAL_COMPLETE** — Обучение завершено

## 🔧 Типы валидации

- **button_click** — Ожидание клика по кнопке

Система полностью адаптивна:
