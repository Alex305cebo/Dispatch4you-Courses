## Модули и их ответственность

### 1. TutorialStateMachine.js

**Ответственность:**
- Управление состояниями (FSM)
- Валидация переходов
- История переходов
- Сохранение/загрузка из localStorage

**Не отвечает за:**
- ❌ UI/визуализацию
- ❌ Данные шагов
- ❌ Игровую логику

**API:**
```javascript
registerState(name, config)
transitionTo(targetState, context)
getCurrentState()
subscribe(callback)
saveToStorage(key)
loadFromStorage(key)
```

---

### 2. MentorshipData.js

**Ответственность:**
- Хранение данных шагов
- Диалоги наставника
- Задачи и подсказки
- UI конфигурация

**Не отвечает за:**
- ❌ Логику переходов
- ❌ Визуализацию
- ❌ Валидацию

**Структура:**
```javascript
export const TUTORIAL_STEPS = [
  {
    id: 'STEP_ID',
    title: 'Заголовок',
    mentorDialog: 'Текст...',
    task: 'Задача...',
    ui: { ... },
    validation: { ... },
    nextSteps: [...]
  }
]
```

---

### 3. TutorialUIController.js

**Ответственность:**
- Создание UI элементов
- Показ/скрытие диалогов
- Подсветка элементов
- Анимации и переходы
- Event Bus для UI событий

**Не отвечает за:**
- ❌ Логику обучения
- ❌ Данные шагов
- ❌ Валидацию

**API:**
```javascript
showMentorDialog(config)
hideMentorDialog()
highlightElement(selector, options)
removeHighlight(selector)
showTooltip(element, message, position)
on(eventName, callback)
```

---

### 4. TutorialManager.js

**Ответственность:**
- Координация всех модулей
- Инициализация системы
- Обработка игровых событий
- Управление жизненным циклом
- Интеграция с игровой логикой

**Не отвечает за:**
- ❌ Детали реализации модулей

**API:**
```javascript
start(forceRestart)
stop()
reset()
skip()
getProgress()
isCompleted()
```

## Паттерны проектирования

### 1. State Machine (FSM)
Управление состояниями обучения через конечный автомат.

### 2. Observer Pattern
Подписка на изменения состояния через `subscribe()`.

### 3. Event Bus
Связь между UI и логикой через события.

### 4. Facade Pattern
TutorialManager как единая точка входа.

### 5. Strategy Pattern
Разные типы валидации (button_click, custom, time_based).

### 6. Data-Driven Design
Шаги обучения определяются данными, а не кодом.

## Зависимости между модулями

```
TutorialManager
    ├── depends on → TutorialStateMachine
    ├── depends on → TutorialUIController
    └── depends on → MentorshipData

TutorialStateMachine
    └── no dependencies (чистый FSM)

TutorialUIController
    └── no dependencies (чистый UI)

MentorshipData
    └── no dependencies (чистые данные)

integration.js
    └── depends on → TutorialManager
```

**Принцип:** Модули низкого уровня не зависят от модулей высокого уровня.
