```typescript
// → автосохранение использует этот слотId
```

```typescript
// → если слот пуст, пробует гибридную загрузку (Firebase + основное хранилище)
```

## Как это работает

### Сценарий 1: Новая игра в слоте
1. Игрок открывает Профиль
2. Кликает на пустой слот
3. Подтверждает "Начать новую игру в Слот X?"
4. `onStartGame(slotId)` → `handleNewGame(slotId)` → `startShift(1, 'Новая смена', slotId)`
5. Игра стартует, `currentSlotId` сохраняется в state
6. Автосохранение использует этот слот

### Сценарий 2: Загрузка из слота
1. Игрок открывает Профиль
2. Кликает на заполненный слот
3. Подтверждает "Загрузить игру из Слот X?"
4. `loadGame(slotId)` загружает данные из `localStorage['dispatcher-save-slot-{slotId}']`
5. Перенаправление на `/game`
6. Игра продолжается с загруженного состояния

### Сценарий 3: Удаление слота
1. Игрок кликает на 🗑️ возле слота
2. Подтверждает "Удалить сохранение из Слота X?"
3. `localStorage.removeItem('dispatcher-save-slot-{slotId}')`
4. Слот становится пустым

## Совместимость

### Старые сохранения
- Основное хранилище `dispatcher-game-save` продолжает работать
- Гибридная система (Firebase + localStorage) работает параллельно
- Если слот пуст, `loadGame()` пробует загрузить из основного хранилища

### Автосохранение
- Если игра запущена через слот → сохраняет в этот слот
- Если игра запущена без слота → сохраняет в основное хранилище
- Оба варианта работают одновременно
