## 🚀 СТАНДАРТНЫЙ ПРОЦЕСС (5 ШАГОВ)

### 1. Собрать проект
```bash
cd DispatcherTraining/adventure
npm run build:web
cd ..
```

### 2. Добавить изменения
```bash
git add -A
```

### 3. Создать коммит
```bash
git commit -m "BETA vX.X.X: Описание изменений"
```

### 4. Запушить в GitHub
```bash
git push origin main
```

### 5. Проверить статус
```
https://github.com/Alex305cebo/Dispatch4you-Courses/actions
```

## ⚡ АВТОМАТИЧЕСКИЙ ДЕПЛОЙ

После `git push` GitHub Actions автоматически:
1. ✅ Получает код

3. ✅ Загружает на Hostinger
4. ✅ Распаковывает файлы
5. ✅ Очищает кеш

**Время**: ~5-10 минут

## 🎯 ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Пример 1: Простой деплой
```
Пользователь: "Деплой"

Kiro выполняет:
1. npm run build:web
2. git add -A
3. git commit -m "Update: Latest changes"
4. git push origin main
5. Создаёт DEPLOY_STATUS.md
6. Сообщает о статусе
```

### Пример 2: Деплой новой версии
```
Пользователь: "Деплой версии 3.5"

Kiro выполняет:
1. Обновляет package.json → "3.5.0-beta"
2. Обновляет menu.tsx → "BETA 3.5"
3. npm run build:web
4. git add -A
5. git commit -m "BETA v3.5.0: New features"
6. git push origin main
7. Создаёт CHANGELOG_v3.5.md
8. Создаёт DEPLOY_STATUS_v3.5.md
9. Сообщает о статусе
```

## 💡 ВАЖНЫЕ ПРАВИЛА

1. **Всегда собирать локально** — проверить что `npm run build:web` работает
2. **Всегда обновлять версию** — при новом релизе
3. **Всегда создавать CHANGELOG** — документировать изменения
4. **Всегда проверять после деплоя** — не считать завершённым пока не проверено
5. **Всегда иметь бэкап** — GitHub Actions создаёт автоматически
