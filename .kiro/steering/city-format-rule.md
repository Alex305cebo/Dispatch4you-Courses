# КРИТИЧЕСКОЕ ПРАВИЛО: Формат городов и штатов

## Обязательный формат для ВСЕГО проекта

**ВСЕ города и штаты ДОЛЖНЫ быть в формате: `Город Штат`**

### Примеры правильного формата:
- ✅ Miami FL
- ✅ Los Angeles CA
- ✅ New York NY
- ✅ Houston TX
- ✅ Phoenix AZ
- ✅ Dallas TX
- ✅ Atlanta GA
- ✅ Chicago IL
- ✅ Denver CO
- ✅ Seattle WA

### Примеры НЕПРАВИЛЬНОГО формата:
- ❌ Miami, FL (запятая)
- ❌ Los Angeles, CA (запятая)
- ❌ Miami (без штата)
- ❌ FL Miami (штат перед городом)
- ❌ Miami Florida (полное название штата)

## Где применять:

1. **Все UI компоненты** - любое отображение городов
2. **Load Board** - маршруты грузов
3. **Truck Detail Modal** - локации траков
4. **Driver Scorecard** - история грузов, предпочтения
5. **City Detail Modal** - информация о городах
6. **Events** - события с локациями
7. **Notifications** - уведомления с городами
8. **AI рекомендации** - маршруты и советы

## Функция cityState()

Используй функцию `cityState()` из `constants/config.ts` для конвертации:

```typescript
import { cityState } from '../constants/config';

// Использование
<Text>{cityState(truck.currentCity)}</Text>
// Результат: "Miami FL"
```

## ВАЖНО:

- Это правило применяется ко ВСЕМУ проекту без исключений
- Всегда используй `cityState()` для отображения городов
- Никогда не добавляй запятые между городом и штатом
- Всегда используй 2-буквенные коды штатов (не полные названия)
