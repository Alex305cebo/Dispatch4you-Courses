# 🚀 Руководство по деплою на Hostinger

## 📋 Что произойдёт после деплоя

После push в репозиторий и деплоя на Hostinger:
- ✅ robots.txt станет доступен
- ✅ sitemap.xml станет доступен
- ✅ Обновлённые мета-теги применятся
- ✅ Структурированные данные добавятся
- ✅ SEO оценка поднимется до 80/100

---

## 🔄 Процесс деплоя

### Вариант 1: Автоматический деплой (GitHub → Hostinger)

```bash
# 1. Добавить все изменения
git add .

# 2. Создать коммит
git commit -m "SEO optimization: add robots.txt, sitemap.xml, meta tags"

# 3. Отправить на GitHub
git push origin main
```

**Hostinger автоматически подхватит изменения через 1-5 минут.**

---

### Вариант 2: Ручная загрузка через FTP/File Manager

Если автодеплой не настроен:

1. **Откройте Hostinger hPanel**
2. **Files → File Manager**
3. **Перейдите в public_html**
4. **Загрузите файлы:**
   - robots.txt
   - sitemap.xml
   - index.html (обновлённый)
   - about.html (обновлённый)

```

3. Вкладка Elements → найдите <head>
4. Проверьте наличие:

```

3. Нажмите "Debug"
4. Нажмите "Scrape Again" (обновить кеш)

## 🔧 Если что-то не работает

### robots.txt не открывается (404)
**Причина:** Файл не загрузился или в неправильной папке

**Решение:**
1. Проверьте что файл в корне: `public_html/robots.txt`
2. Проверьте права доступа: 644
3. Очистите кеш браузера (Ctrl+F5)

### sitemap.xml не открывается (404)
**Причина:** Файл не загрузился

**Решение:**
1. Проверьте что файл в корне: `public_html/sitemap.xml`
2. Проверьте права доступа: 644

### Мета-теги не обновились
**Причина:** Кеш браузера или CDN

**Решение:**
1. Очистите кеш браузера (Ctrl+Shift+Delete)
2. Откройте в режиме инкогнито
3. Подождите 5-10 минут для обновления CDN

### OG теги не отображаются в Facebook
**Причина:** Facebook кеширует старую версию

**Решение:**
1. Откройте Facebook Debugger
2. Нажмите "Scrape Again" несколько раз
3. Подождите 5-10 минут

## 🆘 Частые проблемы

### Проблема: Деплой не происходит
**Решение:**
1. Проверьте подключение к GitHub
2. Проверьте настройки автодеплоя в Hostinger
3. Проверьте логи деплоя в hPanel

### Проблема: Файлы не обновляются
**Решение:**
1. Проверьте что коммит успешно отправлен
2. Проверьте статус деплоя в Hostinger
3. Подождите 5-10 минут
4. Очистите кеш

### Проблема: Ошибки на сайте после деплоя
**Решение:**
1. Проверьте консоль браузера (F12)
2. Проверьте Error Logs в hPanel
3. Откатите изменения если нужно:
   ```bash
   git revert HEAD
   git push origin main
   ```
