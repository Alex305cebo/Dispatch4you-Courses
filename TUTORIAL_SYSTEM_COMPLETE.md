## 🏗️ Архитектура

### Модули и их роли

```
TutorialManager (Оркестратор)
    ├── TutorialStateMachine (FSM логика)
    ├── TutorialUIController (Визуализация)
    └── MentorshipData (Данные)
```

### Паттерны проектирования

1. **State Machine (FSM)** — управление состояниями
2. **Observer Pattern** — подписка на изменения
3. **Event Bus** — связь между модулями
4. **Facade Pattern** — единая точка входа
5. **Strategy Pattern** — разные типы валидации
6. **Data-Driven Design** — данные отдельно от логики

## 🚀 Как использовать

### Шаг 1: Базовая интеграция

Добавь в главный файл игры:

```html
<script type="module">
  import { initializeAll } from './js/tutorial/integration.js';
  initializeAll();
</script>
```

### Шаг 2: Добавь ID к кнопкам

```html
<button id="start-shift-btn">Начать смену</button>
<button id="accept-load-btn">Принять груз</button>
<button id="send-dispatch-btn">Отправить dispatch</button>
<button id="send-pod-btn">Отправить POD</button>
<button id="start-game-btn">Начать игру</button>
```

### Шаг 3: Готово!

Система автоматически:
- Запустит обучение для новых пользователей
- Интегрируется с документами (`.rc-field`)
- Добавит обработчики для кнопок
- Сохранит прогресс

## 🎨 Что можно настроить

### Легко изменить:

✅ **Диалоги наставника** — редактируй `MentorshipData.js`  
✅ **Задачи для игрока** — редактируй `MentorshipData.js`  
✅ **Порядок шагов** — меняй `nextSteps` в `MentorshipData.js`  
✅ **UI элементы** — меняй `ui` конфигурацию в шагах  
✅ **Условия завершения** — меняй `validation` в шагах  
✅ **Стили диалогов** — CSS в `TutorialUIController.js`  

### Требует изменения кода:

⚠️ **Новый тип валидации** — добавь case в `TutorialManager`  
⚠️ **Новый UI компонент** — добавь метод в `TutorialUIController`  
⚠️ **Новая логика переходов** — измени `TutorialStateMachine`
